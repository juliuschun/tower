# Tower

**스스로 도구를 만드는 AI 협업 시스템.**

팀 전체가 명령을 내리고, Tower가 프로세스를 조율합니다. 결과물이 쌓이고, 도구는 정교해지고, 챗봇이었던 것이 팀의 **자가 증식하는 중앙 지능(Central Intelligence)** 이 됩니다.

## 핵심 아이디어

팀은 매일 일합니다. 코드를 짜고, 결정을 내리고, 문서를 쓰고, 문제를 해결합니다.

그런데 그 과정이 실제로 어디에 남습니까? 흩어진 터미널. Slack 스레드. 누군가의 머릿속. 하루가 끝나면 닫히는 Claude 세션.

**Tower는 이걸 바꿉니다.**

AI 기반 업무가 한 곳을 통해 흐르면 — 세션이 기록되고, 결정이 남고, 파일 변경이 추적되고, 메모리가 팀원과 시간을 넘어 유지되면 — 처음으로 가능해지는 것이 있습니다:

우리 회사가 *어떻게* 일하는지 실제로 볼 수 있게 됩니다. 결과물이 아니라, 과정을.

| 도구 | 기록하는 것 |
|------|------------|
| Jira | 티켓 상태 |
| GitHub | 최종 코드 |
| Slack | 메시지 파편 |
| Notion | 정리된 문서 |
| **Tower** | **이 모든 것이 만들어지는 과정** |

---

## 대화에서 협업으로

대부분의 AI 도구는 **1:1 대화** 패러다임에 갇혀 있습니다. 질문하면 답하고, 대화가 끝나면 내일 다시 처음부터.

Tower는 **n:1 협업**을 위해 만들어졌습니다 — 팀 전체가 하나의 AI와 함께 일하고, AI는 배우고, 기억하고, 성장합니다.

| | 기존 AI (1:1) | **Tower (n:1)** |
|---|---|---|
| **누가 쓰는가** | 터미널의 개발자 한 명 | **브라우저의 팀 전체** |
| **무엇이 나오는가** | 사라지는 대화 | **실제 산출물 — 코드, 문서, 결정** |
| **맥락은 어디에** | 한 기기에 잠김 | **공유 팀 메모리** |
| **어떻게 성장하는가** | 정적 — 업데이트를 기다림 | **자가 진화 — 스스로 도구를 만듦** |
| **사람의 역할** | 프롬프트 입력 | **지시, 교정, 조율** |

핵심 차이: **Human in the loop.** 팀은 Tower를 쓰기만 하는 게 아니라 — 만들어갑니다. 교정할 때마다, 결정할 때마다, 워크플로를 실행할 때마다 시스템이 더 정교해집니다. AI가 팀의 프로세스를 흡수하며 동반 성장합니다.

## 1인용 AI의 세 가지 한계

Claude Code는 미친 듯이 강력합니다. 하지만 1인용으로 설계되었습니다. 팀에서는 세 가지 틈이 벌어집니다:

**맥락의 틈.** 팀 아키텍처의 *이유*를 모릅니다. 매 세션이 백지에서 시작됩니다. 한 사람이 배운 것을 아무도 이어받지 못합니다.

**접근의 틈.** 터미널 앱입니다. 기획자, 디자이너, 분석가, 클라이언트는 CLI를 배우지 않을 겁니다. 팀의 대부분이 AI 혁명에서 빠집니다.

**성장의 틈.** 출시 당시의 도구만 쓸 수 있습니다. 팀만의 고유한 워크플로 — 실제로 일하는 방식을 정의하는 것들 — 은 수동으로 남습니다.

Tower는 셋 다 해결합니다.

## Tower가 다른 점

### 🛠 자가 진화하는 스킬 시스템

이것이 Tower의 핵심입니다. 작업을 실행하는 것에 그치지 않고 — **더 잘 실행할 기계를 스스로 만듭니다.**

Tower는 20개 이상의 내장 스킬 — 브레인스토밍, TDD, 디버깅, 코드 리뷰, 기획, UI/UX 디자인 — 과 함께 출발하고, 팀이 일하면서 새로운 스킬을 만들어갑니다. 오늘의 일회성 작업이 내일의 원클릭 스킬이 됩니다.

- **자율 도구 생성** — 팀의 고유한 문제를 위한 스크립트와 워크플로를 자체 설계
- **영속적 숙달** — 한 번 배운 스킬은 저장되어 팀 전체가 사용
- **복리 성장** — 사람의 교정 하나가 이후 모든 실행을 더 정교하게 만듦

팀은 20%가 아닌 80%에서 시작합니다. 그리고 계속 올라갑니다.

### 🧠 중앙화된 팀 브레인

개별 로컬 히스토리는 사일로입니다. Tower는 개인의 작업을 집단 지능으로 바꾸는 3계층 메모리를 갖추고 있습니다:

1. **자동 메모리** — Claude의 네이티브 맥락, 매 대화마다 로드
2. **워크스페이스 메모리** — 팀 전체에 유지되는 공유 결정 + 학습 기록
3. **활동 훅** — 편집, 명령어, 세션 자동 로깅 — 전문 검색 가능

한 사람이 배운 것이 모두를 위한 Claude의 지식이 됩니다.

### 🌐 브라우저 우선, 역할 기반

개발자, 디자이너, 기획자, 분석가 — 터미널 없이 누구나 Claude Code의 풀 파워를 씁니다. 관리자는 전체 접근, 일반 사용자는 가드레일, 모두가 같은 팀 브레인을 공유합니다.

### 📄 즉시 문서 뷰어

Claude에게 보고서, 대시보드, 제안서를 만들어 달라고 하세요 — 내장 뷰어에서 **즉시** 확인할 수 있습니다. 차트가 있는 HTML, 구문 하이라이팅이 적용된 Markdown, 인라인 PDF. 다운로드도, 추가 앱도 필요 없습니다. Notion 같은 느낌 — 다만 모든 것이 AI 생성, git 버전 관리, 검색 가능.

### 🔧 Git 연동

Claude 편집마다 자동 커밋. 모든 변경이 추적되고, attribution이 붙고, 언제든 롤백 가능합니다.

### 📱 모바일 + 음성

폰은 서버의 리모컨입니다. 마이크를 탭하고, 말하면, Claude가 풀 CPU/RAM으로 실행합니다. 지하철에서 세션을 열고, 책상에서 결과를 확인하세요.

```
  당신의 폰 (어디서든)              당신의 서버 (항상 켜짐)
  ┌─────────────────┐               ┌─────────────────────┐
  │                 │── wifi/cell ──▶│  Claude Code         │
  │   음성 + 눈     │               │  풀 CPU/RAM          │
  │                 │◀── 실시간 ────│  Git, 파일, DB       │
  │                 │    결과       │  20+ 스킬            │
  └─────────────────┘               └─────────────────────┘
```

## 플라이휠

```
1일차:    팀이 Tower 사용 시작
1주차:    Claude가 팀 관습을 학습하고, 결정이 쌓이기 시작
1개월차:  신규 입사자 온보딩 → Claude가 이미 프로젝트 맥락을 알고 있음
3개월차:  "왜 이렇게 만들었지?" → 세션 열면 대화가 바로 거기 있음
6개월차:  팀은 더 빠르고, Claude는 더 스마트하고, 일은 보입니다
1년차:    퇴사해도 사라지지 않는 조직 메모리
```

## 기술 스택

| 레이어 | 기술 |
|--------|------|
| **프론트엔드** | React 18 · TypeScript · Vite 6 · Zustand · Tailwind CSS 4 · CodeMirror 6 |
| **백엔드** | Express · TypeScript · tsx watch · WebSocket (ws) |
| **데이터베이스** | SQLite (better-sqlite3) · WAL 모드 · FTS5 전문 검색 |
| **AI 엔진** | Claude Agent SDK (`@anthropic-ai/claude-agent-sdk`) |
| **연동** | MCP 프로토콜 · Git (네이티브) · PWA · chokidar 파일 감시 |
| **인증** | JWT · bcrypt · 역할 기반 접근 (admin / owner / member / guest) |

## 시작하기

```bash
git clone https://github.com/juliuschun/tower.git
cd tower
bash setup.sh    # 모든 것을 설치하고 몇 가지 질문을 합니다
npm run dev      # → http://localhost:32354
```

자세한 설치, 환경 변수, 프로젝트 구조, 배포 옵션은 **[INSTALL.md](INSTALL.md)**를 참고하세요.
