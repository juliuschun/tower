# Tower

**팀의 AI 업무 레이어.**

처음으로, 우리 팀이 무엇을 만들었는지가 아니라 — 어떻게 만들었는지가 보입니다.

---

## 핵심 아이디어

팀은 매일 일합니다. 코드를 짜고, 결정을 내리고, 문서를 쓰고, 문제를 해결합니다.

그런데 그 과정이 실제로 어디에 남습니까? 흩어진 터미널. Slack 스레드. 누군가의 머릿속. 하루가 끝나면 닫히는 Claude 세션.

**Tower는 이걸 바꿉니다.**

AI 기반 업무가 한 곳을 통해 흐르면 — 세션이 기록되고, 결정이 남고, 파일 변경이 추적되고, 메모리가 팀원과 시간을 넘어 유지되면 — 처음으로 가능해지는 것이 있습니다:

우리 회사가 *어떻게* 일하는지 실제로 볼 수 있게 됩니다. 결과물이 아니라, 과정을.

---

## 왜 이게 중요한가

대부분의 도구는 **결과물**을 기록합니다:

| 도구 | 기록하는 것 |
|------|------------|
| Jira | 티켓 상태 |
| GitHub | 최종 코드 |
| Slack | 메시지 파편 |
| Notion | 정리된 문서 |
| **Tower** | **이 모든 것이 만들어지는 과정** |

어떤 도구도 일이 실제로 만들어지는 과정 — 추론, 트레이드오프, 그 결정에 이르기까지의 대화 — 을 기록하지 않습니다.

Tower는 과정을 기록합니다.

팀의 AI 업무가 Tower를 통해 흐르면, 살아있는 기록을 얻게 됩니다:
- **무엇을** 결정했는지 — 그리고 **왜** 결정했는지 (Claude와의 대화가 그 자리에 있습니다)
- **누가** 무엇을 했는지 (사용자 + 세션이 태깅된 git 커밋)
- **지금 무슨 일이 일어나고 있는지** (실시간 활성 세션 가시성)
- **팀이 무엇을 배웠는지** (모든 팀원의 Claude를 더 스마트하게 만드는 공유 메모리)

---

## Claude Code의 문제 (Claude 탓이 아닙니다)

Claude Code는 미친 듯이 강력합니다. 하지만:

**터미널 앱입니다.** 기획자, 디자이너, 분석가, 클라이언트는 CLI를 배우지 않을 겁니다. 절대로.

**한 대의 컴퓨터에 갇혀 있습니다.** 공들여 설정한 스킬, CLAUDE.md, 워크스페이스 맥락 — 모두 한 기기에 잠겨 있습니다.

**공유되지 않습니다.** 세션이 사라집니다. 맥락이 리셋됩니다. 한 사람이 배운 것을 아무도 이어받지 못합니다.

Tower는 이 모든 걸 해결합니다 — 그리고 한 가지를 더합니다.

---

## 작동 방식

```
팀원이 브라우저를 엽니다
      ↓
Claude와 작업합니다 (코드, 문서, 결정, 리서치...)
      ↓
모든 것이 Tower를 통해 흐릅니다
      ↓
세션 기록 · 파일 커밋 · 메모리 업데이트 · 맥락 공유
      ↓
다음 세션은 더 스마트하게 시작됩니다.
다음 팀원은 맥락을 가지고 시작합니다.
다음 결정은 이전 결정 위에 쌓입니다.
```

팀이 Tower를 오래 쓸수록, 더 많이 알게 됩니다. 더 많이 알수록, 더 잘 작동합니다. **플라이휠입니다.**

---

## 대화에서 협업으로

대부분의 AI 도구는 채팅 창을 줍니다. 질문하면 답하고, 대화가 끝나면 내일 다시 처음부터.

Tower는 다른 전제에서 출발합니다: **AI는 챗봇이 아니라 동료여야 한다.**

Tower를 통해 일할 때, 당신은 허공에 메시지를 보내는 게 아닙니다 — 파일을 읽고, 코드를 짜고, 문서를 만들고, 변경을 커밋하고, 배운 것을 기억하는 에이전트를 지휘하는 겁니다. 실시간으로 작업을 보면서 중간에 방향을 바꿀 수 있습니다. 어제 만든 것 위에 오늘을 쌓아갑니다.

같은 것에 새 껍데기를 씌운 게 아닙니다. AI와의 관계 자체를 바꾸는 아키텍처입니다.

| AI와 대화 | AI와 협업 |
|---|---|
| 질문 하나, 답변 하나 | 며칠, 몇 주에 걸친 지속적 세션 |
| 대화마다 맥락 초기화 | 팀 전체에 메모리 축적 |
| 결과물이 클립보드로 | 결과물이 버전 관리되는 파일로 |
| 나만 이득 | 팀 전체가 이득 |
| AI에게 *말한다* | AI와 *함께 만든다* |

Tower는 이 전환을 가능하게 하는 아키텍처입니다 — 브라우저 접근, 지속적 메모리, 파일 시스템, git 연동, 역할 기반 협업, 모바일 접근이 하나로 작동합니다.

## 무엇을 얻는가

### 🌐 브라우저 접근
개발자, 디자이너, 기획자, 분석가 — 터미널 없이 팀 누구나 Claude Code의 풀 파워를 씁니다. 역할 기반 권한으로 관리자는 전체 접근, 일반 사용자는 가드레일.

### 🧠 3계층 팀 메모리

1. **자동 메모리** — Claude의 네이티브 MEMORY.md, 매 대화마다 로드
2. **워크스페이스 메모리** — 팀 전체에 유지되는 공유 결정 + 학습 기록
3. **활동 훅** — 편집, 명령어, 세션 자동 로깅 — 전문 검색 가능

한 사람이 배운 것이 모두를 위한 Claude의 지식이 됩니다.

### 👁 업무 가시성
- 팀 전체 활성 세션을 실시간으로 조회
- 사용자 + 세션 attribution이 붙은 모든 파일 변경 커밋
- `decisions/`의 결정 기록 — 무엇을 했는지의 이유
- 누군가 브라우저를 닫아도 사라지지 않는 세션 히스토리

### 📋 20개 내장 스킬
브레인스토밍, TDD, 디버깅, 코드 리뷰, 기획, UI/UX 디자인 — 미리 설정되어 즉시 사용 가능. 팀이 20%가 아닌 80%에서 시작합니다.

### 🔒 역할 기반 접근
- **관리자**: 전체 워크스페이스 접근, `bypassPermissions` 모드
- **사용자**: `allowed_path` 제한, `acceptEdits` 모드
- **파일 공유**: 내부(팀) 또는 외부(만료되는 공개 링크)

### 🔧 Git 연동
Claude 편집마다 자동 커밋. 모든 변경이 추적되고, attribution이 붙고, 언제든 롤백 가능합니다.

### 📄 내장 문서 뷰어 — 설치 없이 즉시 미리보기

Claude에게 보고서, 대시보드, 제안서, 회의록을 만들어 달라고 하세요 — 결과를 **즉시** 확인할 수 있습니다. 다운로드도, 추가 앱도, "이 파일을 다른 프로그램으로 열어주세요"도 필요 없습니다.

Tower는 HTML, Markdown, PDF 파일을 내장 뷰어에서 바로 렌더링합니다:

- **HTML** — 인터랙티브 대시보드, 스타일링된 보고서, 랜딩 페이지 목업이 차트와 애니메이션까지 그대로 표시됩니다
- **Markdown** — GFM 완전 지원. 구문 하이라이팅, 테이블, 이미지 포함. `.md` 파일이 Notion 페이지처럼 보입니다
- **PDF** — 계약서, 견적서, 스펙 문서 — Tower를 벗어나지 않고 바로 확인

**비개발자에게 왜 중요한가:**

대부분의 AI 도구는 파일을 어딘가 폴더에 만들어 놓습니다. 찾아야 하고, 열 방법을 알아야 하고, 뭔가를 설치해야 할 수도 있습니다. Tower는 이 모든 걸 생략합니다. 파일이 만들어지는 순간 사이드 패널에 바로 뜹니다 — 왼쪽에 미리보기, 오른쪽에 대화.

이 덕분에 워크스페이스가 **Notion이나 공유 위키** 같은 느낌으로 바뀝니다 — 다만 모든 것이 AI가 생성하고, git으로 버전 관리되고, 팀 전체 히스토리에서 검색 가능합니다.

### 📱 모바일 — 어디서든 당신의 AI와 연결

Tower는 모든 브라우저에서 작동합니다. 폰에서 열면, 강력한 가능성이 열립니다:

**음성 우선 워크플로.** 폰 키보드의 마이크 버튼을 누르고 말하세요. 기기의 온디바이스 음성 인식이 텍스트로 바꿔줍니다 — 추가 앱도, 음성 API도, 설정도 필요 없습니다. 동료에게 말하듯 Claude에게 말하면 됩니다. 명령어를 한 글자도 타이핑하지 않아도 되므로, CLI를 절대 배우지 않을 사람에게도 문이 열립니다.

**주머니 속의 서버급 컴퓨팅.** 폰이 연산을 하는 게 아닙니다. 폰은 리모컨입니다. Claude는 서버의 풀 CPU, 메모리, 디스크로 실행됩니다. 데이터 분석, 코드 생성, 문서 작성, 수천 개 파일에 걸친 작업 — 모두 서버 속도로 처리됩니다. 폰의 휴대성과 워크스테이션의 마력을 동시에 얻는 겁니다.

**이동 중에도 세션 생성.** 비행기 대기 중? 지하철? 새 세션을 열고, 지시사항을 음성으로 불러주고, Claude가 작업하게 두세요. 한 번에 다섯 개 세션을 연달아 만들어도 됩니다. 책상에 돌아오면 모든 게 그대로 — 파일 커밋 완료, 맥락 저장, 이어서 작업 가능.

```
당신의 폰 (어디서든)           당신의 서버 (항상 켜짐)
    ┌──────────┐               ┌──────────────────┐
    │  음성    │──── wifi ────▶│  Claude Code      │
    │  입력    │   or LTE      │  풀 CPU/RAM       │
    │  + 눈   │◀─────────────│  Git, 파일, DB    │
    └──────────┘   실시간 결과  │  20개 스킬        │
                               └──────────────────┘
```

이것이 "모바일 AI"의 진짜 의미입니다 — 기능이 빠진 채팅 앱이 아니라, 세계 어디서든 팀의 전체 AI 워크스페이스에 접근하는 것.

## 플라이휠

```
1일차:    팀이 Tower 사용 시작
1주차:    Claude가 팀 관습을 학습하고, 결정이 쌓이기 시작
1개월차:  신규 입사자 온보딩 → Claude가 이미 프로젝트 맥락을 알고 있음
3개월차:  "왜 이렇게 만들었지?" → 세션 열면 대화가 바로 거기 있음
6개월차:  팀은 더 빠르고, Claude는 더 스마트하고, 일은 보입니다
1년차:    퇴사해도 사라지지 않는 조직 메모리
```

## 시작하기

```bash
git clone https://github.com/juliuschun/tower.git
cd tower
bash setup.sh    # 모든 것을 설치하고 몇 가지 질문을 합니다
npm run dev      # → http://localhost:32354
```

자세한 설치, 환경 변수, 프로젝트 구조, 배포 옵션은 **[INSTALL.md](INSTALL.md)**를 참고하세요.

## 포함된 것들

| | |
|---|---|
| **20개 AI 스킬** | 브레인스토밍, TDD, 디버깅, 코드 리뷰, 기획, UI/UX 디자인 등. [`claude-skills/README.md`](claude-skills/README.md) 참고. |
| **3계층 메모리** | 자동 메모리 + 워크스페이스 메모리 + 세션 훅. Claude가 세션을 넘어 기억합니다. [`memory-hooks/README.md`](memory-hooks/README.md) 참고. |
| **워크스페이스 템플릿** | 팀 원칙, 결정 기록, 공유 문서 — `setup.sh`로 자동 생성. |
| **파일 편집기 + 뷰어** | 구문 하이라이팅 편집기 + HTML, Markdown, PDF를 바로 렌더링하는 내장 문서 뷰어. 추가 설치 불필요. |
| **Git 연동** | Claude 편집 시 자동 커밋, 커밋 히스토리, diff 뷰어, 롤백. |
| **관리자 패널** | 사용자 관리, 역할 기반 권한, 사용자별 워크스페이스 제한. |
| **모바일 + 음성** | 반응형 레이아웃, 하단 탭 바, PWA 지원. 폰의 음성 받아쓰기로 Claude에게 지시 — 서버의 풀 컴퓨팅 파워를 어디서든 사용. |
