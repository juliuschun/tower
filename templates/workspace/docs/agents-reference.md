# Workspace Agent Reference

CLAUDE.md에는 항상 필요한 핵심 규칙만 두고, 이 문서에는 필요할 때만 보는 상세 참고 정보를 둡니다.

## 문서화 위치
- 팀 공통 결정: `workspace/decisions/YYYY-MM-DD-title.md`
- 프로젝트 결정: `workspace/projects/<project>/.project/decisions/YYYY-MM-DD-title.md`
- 프로세스/가이드: `workspace/docs/title.md`

## 운영 참고
- 서버/배포/런북 같은 상세 운영 문서는 별도 docs 파일에 둡니다.
- 장문 절차는 CLAUDE.md에 모두 넣지 말고, 필요한 경우 이 문서나 프로젝트 docs를 먼저 확인합니다.

## 응답 원칙 보강
- 설명은 간단명료하게 하고, 필요할 때만 상세 문서를 확인합니다.
- 프로젝트별 규칙이 있으면 프로젝트의 AGENTS.md / CLAUDE.md를 우선합니다.
- 결정이 생기면 문서화 위치를 제안합니다.
