# Tower 제품 지식 (Product Knowledge)

> 이 문서는 AI가 Tower 기능을 정확히 안내하기 위한 참조 문서입니다.
> 사용자가 Tower 사용법, 기능, 스킬 등을 물으면 이 문서를 기반으로 답변하세요.

---

## 1. Tower 개요

Tower는 팀을 위한 AI 워크스페이스입니다. 채팅으로 AI에게 질문하고, 문서를 만들고, 팀과 협업하고, 반복 업무를 자동화할 수 있습니다.

**핵심 공간 3가지:**

| 공간 | 설명 | 접근 |
|------|------|------|
| **세션 (Sessions)** | AI와 1:1 대화. 개인 작업 공간 | 사이드바 → Sessions |
| **채널 (Channels)** | 팀 채팅방. AI도 참여 가능 | 사이드바 → Channels |
| **파일 (Files)** | 공유 파일 탐색기 | 사이드바 → Files |

추가로 헤더의 **칸반 아이콘**을 누르면 태스크 보드가 열립니다.

---

## 2. 세션 (1:1 AI 대화)

- 사이드바에서 `+` 버튼으로 새 세션 생성
- AI와 자유롭게 대화하며 작업 수행
- 프로젝트에 연결하면 해당 프로젝트의 파일과 맥락을 AI가 참조
- 세션 공유: 팀원에게 공유하거나, 외부 링크(읽기 전용)로 공유 가능

---

## 3. 채널 AI (@ai / @task)

채널에서 AI를 호출하는 두 가지 방법:

### @ai — 빠른 질문/답변
```
@ai 이번 분기 매출 데이터 요약해줘
```
- 채널 안에서 바로 AI가 답변
- **대화 맥락 유지**: 같은 채널에서 @ai를 여러 번 쓰면 이전 대화를 기억
- 맥락 초기화: `@ai /reset`

### @task — 본격적인 작업 요청
```
@task 경쟁사 분석 보고서 만들어줘. docs/ 폴더에 저장해줘
```
- 독립 태스크로 생성되어 백그라운드에서 실행
- 칸반 보드에서 진행 상황 확인 가능
- 파일 생성, 검색, 분석 등 복잡한 작업 수행

---

## 4. 시각화 (Dynamic Visual)

AI 답변에 다양한 시각화가 자동으로 렌더링됩니다. 별도 설정 없이 AI가 판단해서 사용합니다.

### 차트
AI에게 데이터를 주면 자동으로 적절한 차트를 그립니다.
- **막대 차트** (bar): 항목 비교
- **꺾은선** (line): 시간 추이
- **영역** (area): 누적 추이
- **파이** (pie): 비율/구성
- **산점도** (scatter): 상관 관계
- **레이더** (radar): 다차원 비교

### 다이어그램
- **Mermaid**: 플로차트, 시퀀스 다이어그램, ER 다이어그램, 마인드맵 등
- **타임라인**: 프로젝트 일정, 로드맵

### 데이터 & 비교
- **데이터 테이블**: 정렬/페이징 가능한 표
- **비교 카드**: 장단점 + 점수 비교
- **트리맵**: 계층 구조 시각화

### 인터랙티브
- **칸반 보드**: 드래그 가능한 카드 보드
- **폼**: 입력 양식
- **스텝 가이드**: 단계별 진행 표시
- **승인 위젯**: 확인/승인 버튼

### 기타
- **지도**: 위치 마커, 경로 표시
- **수식**: LaTeX 수학 수식 ($$...$$ 형식)
- **HTML 샌드박스**: 커스텀 웹 콘텐츠
- **코드 비교**: before/after 차이 표시
- **오디오 플레이어**: 음성 파일 재생

> 사용자가 "차트로 보여줘", "비교표 만들어줘" 등을 요청하면 해당 시각화를 사용하세요.

| 리서치 | `/research` | 여러 소스 병렬 조사 + 종합 |

## 6. 워크스페이스 구조

워크스페이스는 팀의 공유 작업 공간입니다.

```
workspace/
├── guide/                 # ← 이 가이드 문서들
├── principles.md          # 팀 원칙 (수정 불가)
├── decisions/             # 의사결정 기록 (수정 불가, 추가만 가능)
├── docs/                  # 가이드, SOP, 참고 문서
├── projects/              # 프로젝트 폴더 (Tower가 자동 생성)
│   └── <프로젝트>/
│       ├── AGENTS.md      # AI가 읽는 프로젝트 지침
│       └── (프로젝트 파일들)
└── published/             # 완성된 산출물
```

### 핵심 규칙
- **decisions/ 파일은 절대 삭제/수정하지 않습니다.** 변경이 필요하면 새 파일을 만듭니다.
- **프로젝트 = 모든 것의 중심.** 프로젝트에 초대하면 채널, 세션, 파일, AI 맥락 모두 공유됩니다.

## 7. 권한 구조

### 역할
| 역할 | 설명 |
|------|------|
| **Admin** | 전체 관리 (사용자, 설정, 모든 파일) |
| **Operator** | Admin과 동일한 파일 접근, 시스템 설정 제외 |
| **Member** | 기본 역할. 세션/프로젝트 생성 가능, 할당된 폴더만 접근 |
| **Viewer** | 읽기 전용. AI 호출 불가 |

### 채널 역할
| 역할 | 설명 |
|------|------|
| **Owner** | 채널 생성자. 멤버 관리 가능 |
| **Admin** | 채널 관리 권한 |
| **Member** | 채팅 + @ai 사용 가능 |
| **Readonly** | 읽기만 가능, @ai 사용 불가 |

### 파일 접근
- 각 사용자는 접근 가능한 폴더 범위가 정해져 있습니다
- 기본적으로 Member는 `workspace/projects/` 하위만 접근 가능
- Admin이 사용자별로 접근 범위를 조정할 수 있습니다

## 8. 칸반 보드 (태스크 관리)

헤더의 칸반 아이콘을 눌러 태스크 보드를 엽니다.

- **상태**: Todo → In Progress → Done (또는 Failed)
- **생성 방법**: 채널에서 `@task`, 칸반 UI에서 직접, 또는 스케줄 설정
- **진행 추적**: 실시간 진행 상황 확인
- **드래그 앤 드롭**: 상태 변경

---

## 9. 보안 입력

API 키나 비밀번호 같은 민감 정보가 필요할 때:
- AI가 직접 물어보지 않고 **보안 입력 위젯**을 띄웁니다
- 입력한 값은 채팅 기록에 저장되지 않습니다
- `.env` 파일에 직접 저장됩니다

## 10. 자주 묻는 질문

**Q: AI한테 뭘 시킬 수 있어?**
→ 문서 작성, 데이터 분석, 리서치, 번역, 코드 리뷰, 회의록 정리, 견적서 생성, 프레젠테이션 제작, 메일 발송 등 거의 모든 지식 업무를 할 수 있습니다.

**Q: 파일은 어디에 저장돼?**
→ 프로젝트 폴더(`workspace/projects/프로젝트명/`) 안에 저장됩니다. AI에게 "여기에 저장해줘"라고 경로를 지정할 수도 있습니다.

**Q: 이전 대화를 찾고 싶어**
→ `/search` 또는 사이드바 하단의 History를 사용하세요.

**Q: AI가 잘못된 답변을 해요**
→ 맥락을 더 구체적으로 주세요. 프로젝트 폴더 안에서 대화하면 AI가 관련 파일을 참조할 수 있어 정확도가 올라갑니다.

**Q: 여러 사람이 동시에 써도 돼?**
→ 네. 각자의 세션은 독립적이고, 채널에서 함께 대화할 수 있습니다.
