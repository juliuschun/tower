2. Sequence — Inverse Actor

sequenceDiagram
  participant U as User
  participant A as API
  participant D as DB
  U->>A: POST /order
  activate A
  A->>D: BEGIN TX
  alt 재고 있음
    D-->>A: stock=42
    A->>D: INSERT order
    D-->>A: ok
  else 품절
    D-->>A: stock=0
    A-->>U: 409 Conflict
  end
  Note over A,D: RR 격리
  A-->>U: 201 Created
  deactivate A

6. ER Diagram

erDiagram
  USER ||--o{ SESSION : has
  USER ||--o{ MESSAGE : writes
  SESSION ||--o{ MESSAGE : contains
  USER {
    string id PK
    string username
    string role
    bool disabled
  }
  SESSION {
    string id PK
    string user_id FK
    string engine
    timestamp created_at
  }
  MESSAGE {
    string id PK
    string session_id FK
    string role
    text content
  }

7. State Diagram

stateDiagram-v2
  [*] --> Idle
  Idle --> Streaming : user message
  Streaming --> Processing : tool call
  Processing --> Streaming : tool result
  Streaming --> Idle : done
  Streaming --> Error : timeout
  Error --> Idle : retry
  Error --> [*] : abort
