# Proactive Agent — 어디에, 어떻게 보여줄 것인가

**Date**: 2026-04-12
**Status**: draft
**Depends on**: `arch_0412_proactive-agent.md` (구현), `arch_0412_proactive-usage-and-governance.md` (전략)

---

## 진짜 질문

AI가 능동적으로 일을 해서 뭔가를 만들어냈을 때,
그 결과물이 사용자의 일상 흐름 안에 어떻게 자연스럽게 들어와야 하는가?

지금은 벨에 빨간점 하나다. 이건 **알림**이지 **경험**이 아니다.

"AI가 일을 했다"는 사실을 사용자가:
- 놓치지 않고
- 쉽게 발견하고
- 원하면 깊게 들어가고
- 원하지 않으면 치울 수 있어야 한다.

---

## 현재 Tower의 정보 표면 (Surface) 정리

Tower에는 이미 5개의 정보 표면이 있다:

| Surface | 역할 | 특성 |
|---------|------|------|
| **Session List** (사이드바) | 내 모든 대화 | 영구, 프로젝트별 그룹, 라벨 필터 |
| **Inbox** (center panel) | 미읽은 세션 + 최근 유휴 세션 | 카드 형태, 미리보기, 인라인 답장, 일시적 |
| **Bell** (알림) | 이벤트 알림 | 드롭다운, 작은 공간, 스쳐 지나감 |
| **Channel** (Rooms) | 팀 대화 | 그룹, 실시간, AI 멘션 |
| **Kanban** (Task Board) | 태스크 관리 | 상태 기반, 보드 뷰 |

Proactive 메시지의 본질은 **"AI가 만든 unread 세션"**이다.
그렇다면 이미 unread 세션을 보여주는 **Inbox**가 자연스러운 집이다.

---

## 핵심 결정: Inbox가 Proactive의 홈이다

### 왜 Inbox인가

```
Bell        → "알림이 왔다"는 신호 (입구)
Inbox       → "AI가 뭐라고 했는지" 보는 곳 (본체) ← ★ 여기
Session     → "대화를 이어가는" 곳 (깊은 곳)
```

Inbox는 이미:
- ✅ unread 세션을 카드로 보여줌
- ✅ AI 응답 미리보기 텍스트 표시
- ✅ 인라인 답장 가능 (세션 전환 없이)
- ✅ 읽음 처리하면 사라짐 (→ Session List로 이동)
- ✅ 최근 유휴 세션도 아래에 표시

Proactive 세션은 생성 즉시 unread 상태이므로,
**코드 변경 없이도** Inbox에 자동으로 나타난다.

### 하지만 구분이 필요하다

일반 세션과 proactive 세션이 Inbox에 섞이면 구분이 안 된다.
사용자는 "이건 내가 시작한 건가, AI가 건 건가?"를 즉시 알아야 한다.

```
┌─ Inbox ──────────────────────────────────────────────┐
│                                                       │
│  ┌─ 💬 모닝 브리핑 ──────────── 9:03 AM ──────────┐  │
│  │ AI가 시작한 대화                                │  │
│  │                                                 │  │
│  │ "좋은 아침이에요! 어제 3개 태스크가 완료됐고,   │  │
│  │  오늘 예정된 건 배포 파이프라인 검증입니다..."   │  │
│  │                                                 │  │
│  │  [답장 입력...]              [열기] [읽음 처리]  │  │
│  └─────────────────────────────────────────────────┘  │
│                                                       │
│  ┌─ Tower 프로젝트 ──────────── 8:45 AM ──────────┐  │
│  │ (일반 세션 — 내가 시작한 대화)                   │  │
│  │                                                 │  │
│  │ "proactive-agent.ts 파일을 생성했습니다..."     │  │
│  │                                                 │  │
│  │  [답장 입력...]              [열기] [읽음 처리]  │  │
│  └─────────────────────────────────────────────────┘  │
│                                                       │
└───────────────────────────────────────────────────────┘
```

차이점:
- Proactive 카드에 **"AI가 시작한 대화"** 서브타이틀
- 💬 아이콘으로 시각 구분
- 카드 왼쪽에 파란 accent bar (일반 세션은 없음)

---

## 전체 흐름: 표면 간 이동

```
AI가 proactive 실행
  │
  ├──→ 🔔 Bell: "💬 모닝 브리핑" 알림    ← 신호 (놓쳐도 됨)
  │         │
  │         └──→ 클릭 → Inbox로 이동
  │
  ├──→ 📥 Inbox: 카드로 표시             ← 본체 (여기서 소비)
  │         │
  │         ├──→ 인라인 답장 (간단한 응답)
  │         │
  │         └──→ "열기" 클릭 → Session으로 이동
  │
  ├──→ 📋 Session List: "💬 ai" 뱃지    ← 아카이브 (나중에 찾기)
  │
  └──→ 📱 Telegram/Kakao: 외부 푸시     ← Tower 밖에서도 알림
```

각 표면의 역할이 다르다:
- **Bell** = "뭔가 왔다"는 신호. 놓쳐도 Inbox에 남아있다.
- **Inbox** = 실제로 내용을 보고 반응하는 곳. 핵심.
- **Session List** = 대화를 이어가거나 나중에 찾는 곳.
- **External** = Tower에 안 들어와도 아는 곳.

---

## Surface별 상세 설계

### 1. Inbox (핵심 표면)

**현재**: unread 세션을 카드로 보여줌.
**변경**: proactive 세션에 시각적 구분자 추가.

구현:
- `InboxPanel.tsx`에서 `session.label === 'proactive'` 체크
- 카드에 "AI가 시작한 대화" 서브타이틀 + 💬 아이콘 + 파란 accent
- 일반 세션과 동일하게 인라인 답장, 열기, 읽음 처리 가능

**추가 고려: Proactive 섹션 분리**

Inbox가 두 섹션으로 나뉘면 더 명확하다:

```
┌─ Inbox ──────────────────────────────────────────────┐
│                                                       │
│  💬 AI가 건 대화 ─────────────────────────────────── │
│  ├─ 모닝 브리핑 (9:03)                                │
│  └─ 태스크 후속 (8:45)                                │
│                                                       │
│  📨 내 세션 업데이트 ─────────────────────────────── │
│  ├─ Tower 프로젝트 (8:30) — 태스크 완료               │
│  └─ KAP 프로젝트 (어제) — 코드 리뷰                  │
│                                                       │
└───────────────────────────────────────────────────────┘
```

하지만 이건 proactive가 충분히 쌓여서 분리가 의미 있을 때 도입.
처음에는 같은 리스트에 시각 구분만으로 충분하다.

### 2. Session List (사이드바)

**현재**: `💬 ai` 뱃지 표시. (Phase 1에서 구현 완료)
**추가**: Deck 필터에서 `proactive` 라벨로 그룹 필터 가능. (이미 동작)

Session List에서의 proactive 세션 역할:
- Inbox에서 읽음 처리한 후에도 여기서 다시 찾을 수 있음
- 대화를 이어가려면 여기서 세션 선택
- 자동 아카이브 전까지 유지 (미열람 7일, 미답장 14일)

### 3. Bell (알림)

**현재**: 💬 아이콘 + 클릭 시 세션 열기. (Phase 1 완료)
**변경**: 클릭 시 세션 직접 열기 대신 **Inbox로 이동** 옵션 추가.

이유: 벨 드롭다운은 공간이 작다. 미리보기도 한 줄뿐.
Inbox에서 카드로 보는 게 더 자연스러운 경험.

```
Bell 클릭
├── 일반 알림 (task_done, mention) → 기존대로 동작
└── proactive 알림 → Inbox로 이동 (activeView='inbox')
    → Inbox에서 해당 카드가 상단에 표시
```

### 4. External (Telegram/Kakao)

**현재**: 외부 푸시 전송. (Phase 1 완료)
**변경 없음**: Tower 밖에서의 알림은 현재대로 충분.
링크 클릭 → Tower 접속 → Inbox에서 카드 확인.

## 규모가 커지면: 관리 문제

### 자동화가 10개, 20개가 되면?

지금은 수동 API 호출이지만, 앞으로:
- 모닝 브리핑 (매일)
- 태스크 후속 (이벤트)
- 진화 알림 (heartbeat)
- 주간 회고 (매주)
- 비용 리포트 (매주)
- ...

이게 10개 넘어가면 "뭐가 돌고 있는지" 파악이 안 된다.

### 해결: Proactive Dashboard

Settings 안에 "Proactive" 섹션:

```
┌─ Proactive Agent ────────────────────────────────────┐
│                                                       │
│  활성 자동화                                          │
│  ┌────────────────────────────────────────────────┐  │
│  │ ☑ 모닝 브리핑     평일 9시    sonnet   ✏️ 🗑️ │  │
│  │ ☑ 태스크 후속     완료 시     sonnet   ✏️ 🗑️ │  │
│  │ ☐ 주간 회고       금 17시     sonnet   ✏️ 🗑️ │  │
│  │                               [+ 새 자동화]    │  │
│  └────────────────────────────────────────────────┘  │
│                                                       │
│  이번 주                                              │
│  ├─ 실행: 8회  │  열람: 6회 (75%)  │  답장: 3회 (38%)│
│  └─ 토큰: ~16K │  다음 실행: 내일 09:03               │
│                                                       │
│  설정                                                 │
│  ├─ 일일 한도: [5] 회                                 │
│  ├─ Quiet Hours: [22:00] ~ [08:00]                    │
│  └─ 외부 알림: ☑ Telegram  ☐ Kakao                   │
│                                                       │
└───────────────────────────────────────────────────────┘
```

이건 Phase 2. 지금은 API 수동 호출 + 설계서로 충분.

## 멀티유저: 각 사용자는 어떻게 받나

Tower는 멀티유저 환경이다. Proactive는 두 가지 범위가 있다:

### 개인 Proactive (1:1)
- **대상**: 특정 사용자
- **예**: 모닝 브리핑 (내 태스크만), 내 세션 후속
- **전달**: 해당 사용자의 Inbox + Bell + External
- **세션**: 해당 사용자 소유의 private 세션

### 팀 Proactive (1:N)
- **대상**: 프로젝트 멤버 전체
- **예**: 프로젝트 마일스톤, 진화 알림, 장애 알림
- **전달 옵션 A**: 각 멤버에게 개별 세션 생성 (비용 높음, 개인화 가능)
- **전달 옵션 B**: Room에 메시지 게시 (비용 낮음, 공유 맥락)
- **전달 옵션 C**: 하나의 세션 + 멤버 전원에게 알림 (절충)

현재는 개인 Proactive만 구현됨.
팀 Proactive는 기존 Heartbeat L1 (Room 게시)과 역할이 겹침.

**권장 방향**:
- 개인화된 내용 (모닝 브리핑) → 개인 Proactive
- 팀 공유 내용 (진화 알림) → Room 게시 (기존 Heartbeat L1)
- 둘 다 필요 → Room 게시 + 개인 Proactive 동시 실행

## 구현 우선순위 (이 문서 기준)

### 지금 바로 (이미 동작하는 것 활용)

1. ✅ Proactive 세션은 Inbox에 자동으로 나타남 (unread이므로)
2. InboxPanel에 proactive 카드 시각 구분 추가 (accent bar + 서브타이틀)

### 곧 (Phase 1.5)

3. Bell 클릭 시 proactive 알림은 Inbox로 이동
4. 세션 자동 아카이브 (미열람 7일 TTL)

### Phase 2

5. Inbox 섹션 분리 (AI가 건 대화 / 내 세션 업데이트)
6. Proactive Dashboard (Settings 내)
7. 팀 Proactive (Room 연동)

---

## 핵심 원칙

1. **Inbox가 홈이다** — Proactive 메시지의 일차 소비 공간은 Inbox.
2. **Bell은 신호일 뿐** — 벨은 "왔다"를 알리고, 내용은 Inbox에서 본다.
3. **Session은 깊은 곳** — 대화를 이어가려면 세션으로. 아카이브 역할.
4. **섞되 구분하라** — 일반 세션과 같은 리스트에 있되, 시각적으로 즉시 구분.
5. **쌓이지 않게** — TTL 자동 아카이브 + 일일 한도.
6. **개인화 vs 공유** — 개인 내용은 Proactive, 팀 내용은 Room.
