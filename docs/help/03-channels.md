# Channels — 팀 채널

> Channel은 팀원들이 함께 대화하는 공간입니다. 실시간 메시지, AI 호출, 태스크 실행까지 채널 안에서 모두 가능합니다.

---

## Channel이란?

Channel은 여러 사람이 참여하는 팀 대화 공간입니다. Session이 AI와의 1:1 대화라면, Channel은 팀원들과 AI가 함께하는 공유 공간입니다.

프로젝트 논의, 기술 질문, 작업 조율 등 팀 커뮤니케이션에 사용합니다.

---

## Channel 생성

1. Sidebar에서 **Channel** 탭 선택
2. 상단 **+ 버튼** 클릭
3. 채널 이름 입력
4. 생성 완료 — 팀원을 초대하세요

채널 이름은 주제를 명확하게 나타내는 것이 좋습니다. 예: `프론트엔드-리팩토링`, `신규-API-설계`

---

## 참여와 나가기

### 채널 참여

- 채널 목록에서 원하는 채널을 클릭하면 참여할 수 있습니다
- 초대 링크를 통해 참여할 수도 있습니다

### 채널 나가기

- 채널 헤더의 설정 메뉴에서 **Leave** 선택
- 나가도 채널은 삭제되지 않습니다

---

## 실시간 메시지

Channel의 메시지는 실시간으로 동기화됩니다.

- 메시지를 보내면 모든 참여자에게 즉시 표시
- **타이핑 인디케이터**: 누군가 입력 중이면 "○○님이 입력 중..."이 표시됩니다
- 새 메시지가 있으면 Sidebar에서 읽지 않은 메시지 수가 표시됩니다

---

## @ai 멘션 — AI 빠른 답변

채널에서 `@ai`를 입력하고 질문하면, AI가 채널에 직접 답변합니다.

```
@ai 이 에러 메시지가 무슨 뜻이야?
```

### 특징

- **빠른 응답**: 간단한 질문에 즉시 답변
- **팀 전체 공개**: AI의 답변이 채널에 표시되어 팀 전체가 볼 수 있습니다
- **맥락 활용**: 채널의 최근 대화 내용을 참고하여 답변합니다

### 언제 사용하나?

- 짧은 기술 질문
- 코드 스니펫 설명 요청
- 간단한 확인 사항
- 팀원 모두에게 공유할 정보 요청

---

## @task 멘션 — 풀 태스크 실행

채널에서 `@task`를 입력하면 AI가 백그라운드에서 복잡한 작업을 수행합니다.

```
@task 이 모듈의 유닛 테스트 작성해줘
```

### @ai vs @task

| 구분 | @ai | @task |
|------|-----|-------|
| 응답 방식 | 채널에 바로 텍스트 답변 | 백그라운드 태스크로 실행 |
| 소요 시간 | 수 초 | 수 분 ~ 수십 분 |
| 작업 범위 | 질문/답변, 설명 | 코드 작성, 파일 수정, 분석 |
| 진행 상태 | 스트리밍으로 즉시 확인 | Kanban 보드에서 추적 |
| 결과물 | 채팅 메시지 | 파일 변경, 보고서 등 |

### @task 사용 예시

```
@task API 응답 스키마 문서를 자동 생성해줘
@task auth.ts에서 토큰 만료 처리 로직 리팩토링해줘
@task 이번 주 이슈 목록을 정리해서 보고서 만들어줘
```

태스크가 생성되면 Kanban 보드에서 진행 상태를 확인할 수 있습니다.

## Thread — 깊은 토론

채널 메시지에서 Thread를 열면 해당 주제에 대해 별도의 대화를 이어갈 수 있습니다.

### Thread 열기

1. 채널 메시지에 마우스를 올립니다
2. **Reply** 버튼 또는 말풍선 아이콘을 클릭합니다
3. Thread 패널이 열리고 해당 메시지에 대한 대화를 시작합니다

### Thread의 장점

- 채널 메인 대화를 어지럽히지 않고 세부 논의가 가능합니다
- 특정 주제에 집중할 수 있습니다
- Thread 내에서도 @ai, @task를 사용할 수 있습니다

---

## 메시지 답장 (Reply)

특정 메시지에 답장할 수 있습니다.

1. 메시지에 마우스를 올립니다
2. **Reply** 버튼을 클릭합니다
3. 원본 메시지가 인용되면서 답장을 작성할 수 있습니다

답장은 원본 메시지와 연결되어, 대화의 흐름을 쉽게 파악할 수 있습니다.

---

## 멤버 관리

### 멤버 보기

채널 헤더에서 멤버 아이콘을 클릭하면 현재 참여자 목록을 확인할 수 있습니다.

### 멤버 초대

1. 채널 헤더의 멤버 관리 메뉴 열기
2. 초대할 사용자 검색
3. 선택하여 초대 완료

### 멤버 제거

채널 생성자나 Admin은 멤버를 제거할 수 있습니다.

---

## 팁

- **@ai는 가볍게, @task는 무겁게**: 간단한 질문은 @ai, 실제 작업은 @task로 구분하세요.
- **Thread 적극 활용**: 긴 논의는 Thread로 분리하면 채널이 깔끔해집니다.
- **채널 이름 컨벤션**: `프로젝트명-주제` 형식으로 지으면 찾기 쉽습니다. 예: `tower-frontend`, `kap-데이터분석`
- **알림 활용**: 중요한 채널의 메시지를 놓치지 않도록 알림을 확인하세요.
