# Tasks — 칸반 보드

> Kanban 보드에서 AI 태스크를 생성하고, 실행하고, 추적하세요. 드래그앤드롭으로 상태를 바꾸면 AI가 자동으로 작업합니다.

---

## Kanban 보드 열기

Header의 **Kanban 아이콘** (격자 모양)을 클릭하면 태스크 보드가 열립니다.

보드는 세 개의 컬럼으로 구성됩니다:

```
┌─────────────┬─────────────┬─────────────┐
│    Todo      │ In Progress │    Done      │
│             │             │             │
│  대기 중     │  실행 중     │  완료        │
│             │             │             │
└─────────────┴─────────────┴─────────────┘
```

| 컬럼 | 의미 |
|------|------|
| **Todo** | 아직 실행하지 않은 태스크 |
| **In Progress** | AI가 현재 작업 중인 태스크 |
| **Done** | 완료된 태스크 |

---

## 태스크 생성

### 보드에서 직접 생성

1. Kanban 보드 상단의 **+ New Task** 버튼 클릭
2. 태스크 제목과 설명 입력
3. 생성된 태스크가 **Todo** 컬럼에 추가됩니다

### Channel에서 @task로 생성

Channel에서 `@task` 멘션으로도 태스크를 만들 수 있습니다.

```
@task 로그인 페이지 반응형 디자인 구현해줘
```

이렇게 생성된 태스크도 Kanban 보드에 나타납니다.

---

## 태스크 실행

### 드래그앤드롭으로 실행

**Todo** 컬럼의 태스크를 **In Progress**로 드래그하면 AI가 자동으로 작업을 시작합니다.

1. Todo 컬럼에서 태스크 카드를 잡습니다
2. In Progress 컬럼으로 드래그합니다
3. AI가 태스크 내용을 분석하고 작업을 시작합니다
4. 진행 상황이 실시간으로 업데이트됩니다

### 작업 중 확인

In Progress 상태의 태스크 카드를 클릭하면 AI가 현재 어떤 작업을 하고 있는지 확인할 수 있습니다.

---

## 태스크 중단

진행 중인 태스크를 중단하고 싶다면:

1. **In Progress** 컬럼의 태스크를 **Todo**로 다시 드래그합니다
2. AI 작업이 중단되고 태스크가 대기 상태로 돌아갑니다

나중에 다시 In Progress로 이동하면 작업을 재개할 수 있습니다.

---

## 태스크 완료

AI가 작업을 마치면 태스크가 자동으로 **Done** 컬럼으로 이동합니다.

- 완료된 태스크를 클릭하면 결과를 확인할 수 있습니다
- 파일 변경이 있었다면 어떤 파일이 수정되었는지 볼 수 있습니다
- 태스크 완료 시 알림(Notification)이 발생합니다

---

## 스케줄링

태스크를 예약해서 자동 실행할 수 있습니다.

### 일회성 실행

특정 시간에 한 번만 실행되도록 예약합니다.

1. 태스크 카드의 스케줄 설정을 엽니다
2. 실행 날짜와 시간을 선택합니다
3. 설정한 시각에 자동으로 In Progress로 이동하여 실행됩니다

### Cron 반복 실행

정해진 주기로 반복 실행되도록 설정합니다.

1. 태스크 카드의 스케줄 설정을 엽니다
2. **반복 실행**을 선택합니다
3. Cron 표현식 또는 프리셋(매일, 매주 등)을 설정합니다

**사용 예시:**

| 스케줄 | 용도 |
|--------|------|
| 매일 오전 9시 | 일일 리포트 생성 |
| 매주 월요일 | 주간 코드 품질 점검 |
| 매시 정각 | 모니터링 상태 체크 |

## Worktree Cleanup

태스크 실행 중 생성된 Git worktree를 정리하는 기능입니다.

- 완료된 태스크의 임시 worktree가 디스크를 차지할 수 있습니다
- 주기적으로 cleanup하면 디스크 공간을 확보할 수 있습니다
- Kanban 보드 설정에서 cleanup을 실행할 수 있습니다

---

## 태스크 작성 팁

좋은 태스크 설명은 AI의 작업 품질을 높입니다.

### 좋은 예

```
auth.ts의 refreshToken 함수에서 토큰 만료 시
자동 갱신 로직을 추가해줘. 실패하면 로그아웃 처리.
```

- 구체적인 파일명과 함수명 포함
- 원하는 동작을 명확하게 설명
- 예외 처리 방향도 명시

### 나쁜 예

```
로그인 고쳐줘
```

- 어떤 파일인지 알 수 없음
- 무엇을 고치는지 불명확
- 기대 결과가 없음

---

## 팁

- **작은 단위로 분리**: 큰 작업은 여러 태스크로 나누세요. AI가 더 정확하게 작업합니다.
- **스케줄 활용**: 반복되는 점검 작업은 Cron으로 자동화하세요.
- **완료 결과 확인**: Done 컬럼의 태스크를 꼭 열어보고 결과물을 검증하세요.
- **Channel 연동**: 채널에서 @task로 빠르게 태스크를 만들고, Kanban 보드에서 관리하세요.
