# Sessions — AI 대화

> Session은 Claude AI와의 1:1 대화 공간입니다. 질문하고, 코드를 작성하고, 파일을 분석하고, 복잡한 작업을 요청하세요.

---

## Session이란?

Session은 AI와 나만의 대화 스레드입니다. Channel이 팀 전체가 보는 공간이라면, Session은 개인 작업 공간입니다.

각 Session은 독립적인 대화 맥락을 갖습니다. 여러 Session을 동시에 열어두고 주제별로 관리할 수 있습니다.

---

## 새 Session 만들기

### 기본 생성

1. Sidebar에서 **Sessions** 탭 선택
2. 상단 **+ 버튼** 클릭
3. 새 Session이 열리고 입력창에 포커스

### Project 내 생성

Project를 선택한 상태에서 Session을 만들면 해당 프로젝트에 소속됩니다. 프로젝트의 AI 설정(AGENTS.md, CLAUDE.md)이 자동으로 적용됩니다.

---

## 메시지 보내기

| 동작 | 단축키 |
|------|--------|
| 메시지 전송 | **Enter** |
| 줄바꿈 (개행) | **Shift + Enter** |
| 스트리밍 중단 | **Escape** |
| 큐 취소 | **Escape** |

### Streaming

메시지를 보내면 AI가 실시간으로 응답을 스트리밍합니다. 글자가 하나씩 나타나는 것을 볼 수 있습니다.

### 메시지 큐잉

AI가 응답 중일 때 추가 메시지를 보내면 **큐(대기열)**에 들어갑니다. 현재 응답이 끝나면 큐에 쌓인 메시지가 순서대로 전송됩니다.

- 큐에 들어간 메시지는 입력창 위에 표시됩니다
- **Escape**를 누르면 큐에 있는 메시지를 취소할 수 있습니다
- 스트리밍 중 Escape를 누르면 현재 응답이 중단됩니다

---

## Slash Commands

입력창에 `/`를 입력하면 사용 가능한 명령어 목록이 나타납니다.

Slash command를 사용하면 특정 작업을 빠르게 수행할 수 있습니다. 목록에서 원하는 명령을 선택하거나 이름을 타이핑해서 필터링하세요.

---

## 파일 첨부

Session에서 파일을 AI에게 보낼 수 있습니다.

### 첨부 방법

| 방법 | 설명 |
|------|------|
| **드래그앤드롭** | 파일을 입력창 위로 드래그 |
| **클립보드 붙여넣기** | 스크린샷이나 이미지를 Ctrl+V |
| **File Tree에서 드래그** | 왼쪽 파일 트리에서 파일을 입력창으로 |

첨부된 파일은 메시지와 함께 AI에게 전달됩니다. 이미지, PDF, 코드 파일 등 다양한 형식을 지원합니다.

## Session 관리

### 이름 변경

Session 목록에서 세션 이름을 클릭하면 이름을 바꿀 수 있습니다. 대화 주제에 맞는 이름을 붙이면 나중에 찾기 쉽습니다.

### 즐겨찾기 (Pin)

자주 사용하는 Session은 즐겨찾기에 추가하세요.

- Session 우클릭 → **Pin** 선택
- Sidebar 하단 **Pins**에서 모아보기

### 삭제

더 이상 필요 없는 Session은 삭제할 수 있습니다.

- Session 우클릭 → **Delete** 선택
- 삭제된 Session은 복구할 수 없습니다

### Project 간 이동

Session을 다른 프로젝트로 옮길 수 있습니다.

- Session 우클릭 → **Move to Project** 선택
- 대상 프로젝트를 선택하면 이동 완료

## Turn Metrics

각 AI 응답 하단에 Turn metrics가 표시됩니다.

| 항목 | 설명 |
|------|------|
| **토큰 수** | 이번 턴에서 사용한 입력/출력 토큰 |
| **모델** | 응답에 사용된 AI 모델 이름 |
| **소요시간** | 응답 생성에 걸린 시간 |

토큰 사용량을 모니터링하면 비용을 효율적으로 관리할 수 있습니다.

---

## Draft 자동 저장

입력창에 타이핑한 내용은 세션별로 자동 저장됩니다.

- 다른 Session으로 전환해도 작성 중이던 내용이 보존됩니다
- 브라우저를 닫았다 열어도 Draft가 남아 있습니다
- 메시지를 전송하면 해당 Session의 Draft가 초기화됩니다

실수로 탭을 닫거나 다른 세션으로 이동해도 작성 중이던 긴 메시지를 잃어버리지 않습니다.

---

## 팁

- **주제별 Session 분리**: 하나의 Session에 여러 주제를 섞지 마세요. AI가 맥락을 더 잘 이해합니다.
- **Project 활용**: Session을 프로젝트에 소속시키면 AI가 프로젝트 설정을 자동으로 참조합니다.
- **큐잉 활용**: AI 응답을 기다리지 말고 후속 질문을 미리 보내두세요. 순서대로 처리됩니다.
- **파일 첨부 적극 활용**: "이 코드 분석해줘"보다 파일을 직접 첨부하면 더 정확한 답변을 받을 수 있습니다.
