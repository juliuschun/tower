# 단축키 & 꿀팁

> Tower를 더 빠르게 쓰는 방법. 키보드 단축키와 생산성 팁을 정리했습니다.

---

## 채팅 단축키

Session과 Channel 모두에서 사용할 수 있습니다.

| 단축키 | 동작 |
|--------|------|
| **Enter** | 메시지 전송 |
| **Shift + Enter** | 줄바꿈 (개행) |
| **Escape** | 스트리밍 중단 / 큐 취소 |
| **/** | Slash command 메뉴 열기 |
| **Ctrl + V** | 클립보드 이미지 붙여넣기 (파일 첨부) |

### 메시지 입력 상세

- **Enter**만 누르면 바로 전송됩니다
- 여러 줄 메시지를 작성하려면 **Shift + Enter**로 줄바꿈하세요
- AI가 응답 중일 때 **Escape**를 누르면 스트리밍이 중단됩니다
- 큐에 메시지가 있을 때 **Escape**를 누르면 대기 중인 메시지가 취소됩니다

## File Tree 단축키

| 단축키 | 동작 |
|--------|------|
| **우클릭** | 컨텍스트 메뉴 (새 파일, 이름 변경, 삭제 등) |
| **Enter** | 이름 변경 모드에서 이름 확정 |
| **Escape** | 이름 변경 취소 |
| **클릭** | 파일 열기 / 폴더 확장-축소 |

## 드래그앤드롭

Tower에서 드래그앤드롭이 지원되는 곳들입니다.

| 출발지 | 목적지 | 동작 |
|--------|--------|------|
| 외부 파일 | File Tree 폴더 | 파일 업로드 |
| 외부 파일 | 채팅 입력창 | 메시지에 파일 첨부 |
| File Tree 파일 | 채팅 입력창 | 메시지에 파일 첨부 |
| Kanban 카드 | 다른 컬럼 | 태스크 상태 변경 |

### 드래그앤드롭 팁

- **여러 파일 동시 업로드**: 탐색기에서 여러 파일을 선택해서 한번에 드래그하세요
- **Kanban 상태 변경**: Todo → In Progress로 드래그하면 AI가 자동으로 태스크를 실행합니다
- **File Tree → 입력창**: AI에게 특정 파일을 분석해달라고 할 때 가장 빠른 방법입니다

## 꿀팁 모음

### 1. 큐잉으로 연속 질문

AI 응답을 기다리지 마세요. 스트리밍 중에도 다음 질문을 바로 보내면 큐에 쌓입니다. AI가 현재 응답을 마치면 자동으로 다음 질문을 처리합니다.

```
[첫 번째 질문 전송]
→ AI 스트리밍 중...
[두 번째 질문 전송] → 큐에 대기
[세 번째 질문 전송] → 큐에 대기
→ 첫 번째 응답 완료
→ 두 번째 질문 자동 전송
→ ...
```

여러 관련 질문이 있을 때 시간을 크게 절약할 수 있습니다.

### 2. 프로젝트별 Session 정리

Session이 많아지면 프로젝트별로 정리하세요.

- 프로젝트를 선택한 상태에서 Session을 생성하면 자동 소속
- 기존 Session은 우클릭 → Move to Project로 이동
- 프로젝트 필터로 관련 Session만 모아보기

### 3. AI에게 시각화 요청하기

데이터가 있으면 항상 시각화를 요청해보세요.

- "이 데이터를 차트로 보여줘" → 자동으로 chart 블록 생성
- "아키텍처를 다이어그램으로" → mermaid 블록 생성
- "비교표로 정리해줘" → datatable 또는 comparison 블록 생성

자세한 내용은 [Visual Blocks](./06-visual-blocks.md)를 참고하세요.

### 4. Channel에서 AI 활용

모든 팀원이 볼 수 있는 공간에서 AI를 사용하면 지식이 자연스럽게 공유됩니다.

- `@ai` — 빠른 질문/답변 (팀 전체 공유)
- `@task` — 무거운 작업 실행 (결과를 팀이 공유)

### 5. Pin으로 빠른 접근

자주 열어보는 파일, 중요한 Session을 Pin해두세요.

- 파일 Pin → Sidebar Pins에서 한 클릭으로 열기
- Session Pin → 중요한 대화 빠르게 접근
- 핵심 파일 5-6개만 Pin해도 작업 속도가 크게 올라갑니다

### 6. Draft 안심하고 활용

입력 중 Session을 전환해도 작성 중이던 내용이 사라지지 않습니다.

- Session A에서 긴 메시지 작성 중
- Session B로 전환해서 다른 작업
- 다시 Session A로 돌아오면 작성 중이던 내용 그대로

### 7. 태스크를 작게 분리

AI 태스크는 작을수록 정확합니다.

```
나쁜 예: "이 앱 전체 리팩토링해줘"
좋은 예: "auth.ts의 refreshToken 함수에서 에러 핸들링 추가해줘"
```

구체적인 파일명, 함수명, 원하는 동작을 포함하세요.

### 8. 모델 전략적 사용

모든 질문에 최고 모델을 쓸 필요는 없습니다.

| 상황 | 추천 |
|------|------|
| 간단한 질문, 번역 | 빠른 모델 |
| 복잡한 코드 작성 | 고성능 모델 |
| 대량 데이터 분석 | 고성능 모델 |
| 문서 요약 | 빠른 모델 |

### 9. 숨김 파일 체크

설정 파일을 찾을 수 없다면 숨김 파일 토글을 확인하세요. `.env`, `.gitignore`, `.eslintrc` 같은 파일들은 기본적으로 숨겨져 있습니다.

### 10. 경로 복사 활용

File Tree에서 파일 우클릭 → 경로 복사 → 채팅에 붙여넣기. AI에게 정확한 파일을 알려줄 때 유용합니다.

```
이 파일 분석해줘: /workspace/projects/tower/packages/backend/services/auth.ts
```

경로를 직접 타이핑하는 것보다 정확하고 빠릅니다.
